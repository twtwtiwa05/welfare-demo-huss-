# 복지망 잔여 사각지대 발굴·모니터링 AI 에이전트 — 데모

공모전 발표용 **인터랙티브 웹 데모**입니다. 실제 행정 데이터·실서비스가 아니라,
**합성 데이터**로 "시스템이 어떻게 작동하는가"를 단계별로 보여주는 파이프라인 시각화 데모입니다.

> 데이터로는 이기지 않는다 · 해석·특화·활용도로 보완한다 · 최종 판단은 사람

---

## 4대 설계 원칙 (코드 전반에서 강제)

1. **위험 점수는 투명한 가중합 공식이 계산한다.** 신경망/LLM이 점수를 만들지 않는다.
   슬라이더로 입력을 바꾸면 즉시 재계산되는 자바스크립트 함수다(`src/lib/scoring.ts`). 블랙박스 없음.
2. **LLM은 "이미 나온 점수와 신호"를 받아 자연어 근거·권고만 생성한다.** 위험 여부·점수를 판단하지 않는다.
3. **"진짜 계산"과 "시뮬레이션(미리 정해둔 값)"을 화면에 명확히 라벨링**한다(`SimBadge`, `ProofBoundary`).
4. **최종 개입(전화/방문) 결정은 사람이 한다.** AI는 발굴·근거·우선순위까지만. UI엔 "자동 처리" 버튼이 없다.

## 실행법

```bash
npm install        # 의존성 설치 (React 18 / Tailwind v3 / Recharts 2.15 고정)
npm run dev        # 개발 서버 (http://localhost:5173)
npm run build      # 프로덕션 빌드 → dist/
npm run preview    # 빌드 결과 미리보기
npm run gen:data   # 합성 데이터 재생성 (시드 고정, 재현 가능)
```

> **LLM 라이브 호출 없음.** 근거 생성은 결정론적 시뮬레이션(`src/lib/reason.ts`)으로 구현했습니다.
> 백엔드·API 키가 필요 없어 순수 정적 SPA로 동작합니다(발표 중 네트워크/키 사고 위험 0).
> 실서비스에서는 이 자리에 LLM(예: Haiku 4.5급)이 들어갑니다 — 화면에 그 경계를 라벨로 표기했습니다.

## 배포법 (정적 호스팅 어디든)

1. GitHub에 푸시
2. [vercel.com](https://vercel.com) → New Project로 레포 import (환경변수 **불필요**)
3. Deploy → `https://<프로젝트명>.vercel.app` 공개 URL 생성. 이후 푸시 시 자동 재배포.

Netlify·GitHub Pages·Cloudflare Pages 등 어떤 정적 호스팅에도 `dist/`를 올리면 됩니다.

## 화면 구성

### 탭 1 · 파이프라인 (5단계)
| 단계 | 내용 | 증명 |
|---|---|---|
| ① 원시 신호 | 합성 가구의 신호(전력·진료·우편·통신·고립) 표시 | 데이터 표시 |
| ② 정합·중복제거 | 행복e음 기포착 가구 제거 → '잔여'만 분석 | **보완 레이어** (플래그는 시뮬) |
| ③ 투명 점수 | 가중합 공식으로 점수 산출. **슬라이더로 실시간 재계산 + 집단 모형 토글** | **100% 진짜 · 설명가능성 + ② 집단특화** |
| ④ AI 근거 | 점수·신호를 받아 자연어 근거·권고·반대근거·출처 생성 | 시뮬 (역할 분담 명시) |
| ⑤ 우선순위 | 임계값 방식 vs 조합 방식 대조 + 거짓음성 지표 + 순환 자백 | **극복 ①② + 정직성** |

### 탭 2 · 활용도 ③ (Before/After)
같은 잔여 명단을 좌(점수만 있는 무미건조한 명단) / 우(근거·권고·급상승 배지가 붙은 행동 가능한 카드)로 대조.
**"데이터는 똑같습니다. 바뀌는 건 담당자가 행동할 수 있느냐입니다."** — 기획서가 말한 유일한 진짜 차별점(③).

### 탭 3 · 대시보드
행정동 히트맵 · 이번 주 급상승 TOP · 고위험 후보 리스트(등록 외 필터) · 케이스 상세(신호·근거·시계열 + 조치 **기록**).

## 발표 시연 시나리오 (대본)

1. **(증명 경계 패널)** "먼저 이 데모가 증명하는 것과 증명하지 않는 것을 분명히 합니다. 합성 데이터라 발굴 정확도는 증명하지 않습니다."
2. **(② 중복제거)** "행복e음이 이미 18명을 잡았습니다. 우리는 거기서 손을 떼고, 잔여 52명만 봅니다." → '중복제거 실행' 클릭.
3. **(③ 점수)** "잔여 후보 중 한 가구. 점수가 왜 이렇게 나오는지 보겠습니다." → 슬라이더로 전력을 낮춰 점수·막대·색·판정이 함께 바뀌는 것 시연. "미리 정해둔 화면이 아닙니다."
4. **(③ 집단 토글)** "같은 가구를 중장년 특화 모형으로 보면 점수가 올라갑니다. 전국 범용 모형이 놓치는 지점입니다(②)."
5. **(④ 근거)** "이 점수에 LLM이 근거를 붙입니다. **점수는 LLM이 만든 게 아닙니다.** 그리고 반대 근거·불확실성도 함께 답니다."
6. **(⑤ 대조)** "임계값 방식이라면 이 케이스들을 놓쳤습니다 — 임계 미달 조합이라서요. 단, 합성 데이터라 '몇 배'는 말하지 않습니다."
7. **(활용도 ③)** "왼쪽은 명단, 오른쪽은 행동. 데이터는 같습니다." ← **클라이맥스**
8. **(대시보드)** "급상승 가구부터, 이 근거로. **하지만 방문 결정은 사람이 합니다** — 여기선 조치를 '기록'만 합니다."

발표 직전: 전체화면(F11) + 브라우저 확대(Ctrl++)로 시인성 확인.

## 한계 명시 (기획서 5·6장과 동일 톤)

- **합성 데이터라 발굴 성능 증명이 아니다.** 정답(`groundTruthRisk`)을 우리가 설계했으므로 순환이다. 이 데모가 증명하는 것은 파이프라인·근거 리포트·대시보드 UX의 작동이다.
- **행복e음 기포착 플래그(`haengbokFlagged`)는 시뮬레이션**이다. 실서비스에선 연동으로 받는다.
- **실서비스는 법적 근거·지자체 위임·실데이터 연동**이 전제다(개인 식별 결합은 임의로 넘을 수 없는 선).
- 유형 A(등록–실거주 불일치)는 실거주 확인 데이터 없이는 추정에 그친다.
